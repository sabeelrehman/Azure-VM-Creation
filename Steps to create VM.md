# Step 2 - Navigate to the Virtual machines service by clicking the service on the homepage or via the search bar. Start creating a VM by clicking Create → virtual machine at the top bar or by clicking the Create button at the center of the page.
<img width="2402" height="1287" alt="Creating the VM" src="https://github.com/user-attachments/assets/12e3483a-57d6-49ff-9f58-5f397231f345" />

# Step 3 - Configuring the Virtual Machine 
<img width="1405" height="923" alt="VM Configuration" src="https://github.com/user-attachments/assets/0b41790c-620c-496e-a90e-ab5c42c6fa54" />

# Step 4 - Setting the Virtual Machine Size - Navigate to Size. The recommended sizes will appear as not allowed. Click “See all sizes” and choose one of the three permitted VM sizes: B1ls, B1ms, or B1s.
<img width="3320" height="1235" alt="Setting VM Size" src="https://github.com/user-attachments/assets/b88663ba-8716-4e74-ae18-607f24fca46d" />

# Step 5 - Setting the Virutal Machine Image - Selected one of the following: Canonical (Ubuntu), Debian or AlmaLinux (no Pro versions)
<img width="1227" height="1227" alt="Configuring image for VM" src="https://github.com/user-attachments/assets/68c69a97-a490-473f-8bf2-80703b888cd8" />

# Step 6 - Setting VM Disk Type - Select the OS disk type - choose either Standard HDD or Standard SSD only.
<img width="1562" height="1235" alt="Setting VM Disk Type" src="https://github.com/user-attachments/assets/5b02af6a-ee01-4436-be88-41323f8ecba9" />

# Step 7 - Configuring the Virtual Network - Just give the virtual network a unique name
<img width="1313" height="1206" alt="Creating a Virtual Network" src="https://github.com/user-attachments/assets/6456fe84-2d36-4c31-8f48-a75cd89db4e4" />

# Step 8 - Once done, hit Review + Create for the Virtual Machine you just configured. Wait for the “Validation passed” message to show up, then review if all configurations are correct. Once done, click Create.
<img width="1650" height="1242" alt="Deploy VM" src="https://github.com/user-attachments/assets/f433437b-b74c-4528-9ee1-cbb0b116457c" />

# Step 9 - A prompt will show up asking you to generate a new key pair. Select the “download private key to create resource” button. Then save the pem key in your desired location locally.
<img width="2147" height="1205" alt="New Key Pair Generation" src="https://github.com/user-attachments/assets/f82a658d-c80f-4af2-8fa6-570e73007b53" />

# Step 10 - Wait for the deployment to be successful. Once done, check the new VM that you created.
